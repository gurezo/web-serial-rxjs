# Documentation Architecture

This document defines the canonical structure, responsibilities, paths, URL rules, and generated-artifact policy for `@gurezo/web-serial-rxjs` documentation.

Parent issue: [#453](https://github.com/gurezo/web-serial-rxjs/issues/453)  
Defined by: [#454](https://github.com/gurezo/web-serial-rxjs/issues/454)

## Logical structure

```text
Documentation
├── Guide
│   ├── 日本語 (hand-written source → static HTML output)
│   └── English (hand-written source → static HTML output)
└── API Reference
    └── English / TypeDoc (TypeScript JSDoc → generated HTML)
```

## Responsibilities

### Guide (Japanese and English)

| Topic | Description |
| --- | --- |
| Installation | Package install and peer dependency (`rxjs`) |
| Quick Start | Shortest path to an open port and subscriptions |
| Connection flow | Basic connect / disconnect / close workflow |
| Lifecycle | `state$` as the canonical lifecycle source |
| Error handling | `errors$` for fatal and non-fatal errors |
| Read / write | Sending and receiving data |
| Advanced usage | Line framing, request/response-style flows, recovery |
| Migration | Links to v2 → v3 and v1 → v2 migration guides |
| Concepts | Supplemental tables and diagrams (formerly `API_REFERENCE.md`) |

Guide content is **hand-written Markdown** in two languages. It explains *how to use* the library.

### API Reference (English only)

| Topic | Description |
| --- | --- |
| Exported symbols | Classes, interfaces, types, enums, functions |
| Public API surface | Properties, methods, parameters, return types |
| API contracts | `@deprecated`, `@throws`, error codes, emitted errors |
| Source of truth | English TypeScript doc comments in `packages/web-serial-rxjs/src/` |

API Reference is **generated by TypeDoc** from English JSDoc. It documents *what the public API is*. A Japanese API Reference is **not** generated.

### Naming clarification

| Name | Role |
| --- | --- |
| `API Reference` (this document) | TypeDoc output under `docs/api/` |
| `concepts.md` (Guide) | Former `API_REFERENCE.md` — conceptual supplement, not generated API docs |

## URL path rules

| Logical path | GitHub Pages (current) | Firebase (#151 planned) |
| --- | --- | --- |
| Site root | `https://gurezo.github.io/web-serial-rxjs/` | `https://gurezo.net/web-serial-rxjs/` |
| Japanese Guide | `/guide/ja/` | `/web-serial-rxjs/guide/ja/` |
| English Guide | `/guide/en/` | `/web-serial-rxjs/guide/en/` |
| API Reference | `/api/` | `/web-serial-rxjs/api/` |

- Prefer relative links within generated HTML so hosting `base` path changes are handled in #151 / #458.
- `packages/web-serial-rxjs/package.json` `homepage` will be updated in #458.

## Generated files policy

| Path | Git tracking |
| --- | --- |
| `packages/web-serial-rxjs/docs/guide/**` | Yes (hand-written source) |
| `docs/api/**`, `docs/guide/**` | No (CI-generated artifact) |
| `docs/.gitignore` | Yes (ignores all generated content) |

- Local generation: `pnpm run docs` (TypeDoc outputs to `docs/api/`).
- Publishing: `.github/workflows/deploy-docs.yml` uploads `./docs` as the Pages artifact.
- **Do not edit** files under root `docs/` except `docs/.gitignore`.
